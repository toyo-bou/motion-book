# 文字の余白 — motion-book

読める文字の面を、移動するかたちが空白として横切るモーションブック。画面中央の羊皮紙風パネルに詩的な日本語テキストが並び、手続き的に生成された魚がその上を泳ぐ。文字は魚を避けるようにリアルタイムで再配置され、動く余白が生まれる。操作は不要で、眺めるだけの生成的な体験として動作する。

## 仕様

### ビジュアルと画面構成

- 全画面 Canvas 2D 描画。デバイスピクセル比 (DPR) に対応し高解像度ディスプレイでも鮮明に表示
- 背景: 暗い木目調のグラデーションにラジアルグロー、ノイズテクスチャ、ランダムな染みを重ねる
- パネル: 暖色系の紙グラデーション、ノイズ、ビネット、水平方向の繊維線、角丸。アスペクト比は 0.62 を基準にポートレート／ランドスケープで調整
- 画面右下に情報バッジ（FPS・表示文字数）を常時表示。グラスモーフィズム風の半透明デザイン

### テキスト配置

- フォントサイズはパネル面積に応じて 14–22 px の範囲で自動算出し、表示目標文字数 600 に近づける
- パネル内部をグリッド分割し、CJK サンプル文字の実測値（幅倍率 1.28、行高倍率 1.72）で列数・行数を決定
- スロット割り当て時に、魚が占有する領域と余裕分（全体の約 6%）を差し引く
- テキスト再配置には指数イージング（平滑化係数 11）を使用し、文字が滑らかに新しい位置へ移動
- 各スロットの衝突判定は、魚の体節ごとの回転楕円と三角形（頭部・尾部・背びれ）で行う

### 魚のアニメーション

- `SegmentedFish` クラス: 体節 7 個＋頭部＋尾びれで構成
- ランダムな目標点を選択し、ステアリング力で旋回しながら追従するターゲットシーキング方式
- 速度は正弦波で振動（maxSpeed 42–78 px/s、パネルサイズに応じて変動）
- 体幹に正弦波うねりを適用（振幅 6–11 px、速度 1.7–2.3、セグメントごとに位相をずらす）
- 移動履歴を配列で保持し、体節位置を履歴上の距離で補間することで滑らかな曲線を実現
- パネル境界では進行方向を反射し、旋回速度を減衰させて自然に跳ね返る
- 描画: ベジェ曲線による滑らかな体輪郭、半透明の青みがかった塗り、ストローク、背びれ、目

### 描画パイプライン

フレームごとの処理順:

1. FPS カウンタ更新
2. `fish.update(dt, timestamp)` — 物理演算・履歴・体節更新
3. `updateTextFlow(dt)` — 空きスロット判定と文字位置のイージング
4. `drawBackground()` — 背景塗りつぶし＋テクスチャ
5. `drawPanel()` — 影・紙テクスチャ・ウォッシュグラデーション・枠線
6. `drawText()` — 各文字をイージング後の座標に描画
7. `fish.draw(ctx)` — 体・ひれ・目・ハイライト

リサイズ時は 180 ms のデバウンスを経てシーン全体を再構築（テクスチャ再生成、魚の再インスタンス化）。

### 定数一覧

| 定数 | 値 | 説明 |
|---|---|---|
| `PRIMARY_FONT_NAME` | `'Kiwi Maru'` | 本文フォント |
| `MIN_FONT_SIZE` / `MAX_FONT_SIZE` | 14 / 22 | フォントサイズ範囲 (px) |
| `LINE_HEIGHT_RATIO` | 1.72 | 行高さ倍率 |
| `CJK_WIDTH_RATIO` | 1.28 | 全角文字幅倍率 |
| `TARGET_VISIBLE_CHARS` | 600 | 表示目標文字数 |
| `TEXT_FLOW_SMOOTHING` | 11 | テキスト移動の平滑化係数 |
| `RESIZE_DEBOUNCE_MS` | 180 | リサイズ検知の遅延 (ms) |
| `FONT_LOAD_TIMEOUT_MS` | 3000 | フォント読み込みタイムアウト (ms) |

## 技術スタック

- **ビルド**: Vite 5.4.21
- **テキスト計測**: @chenglou/pretext ^0.0.3
- **描画**: Canvas 2D API（フレームワーク不使用、vanilla JavaScript）
- **フォント**: Google Fonts — Kiwi Maru（本文）、M PLUS 1 Code（情報バッジ）
- **ローカルサーバ**: Node.js 組み込み `http` モジュール (`scripts/serve-dist.mjs`)

## ファイル構成

```
motion-book/
  index.html              … エントリポイント
  main.js                 … 描画・アニメーション・テキスト配置
  style.css               … 全画面 Canvas と情報バッジのスタイル
  package.json            … 依存関係と npm スクリプト
  scripts/
    serve-dist.mjs        … 本番ビルド用ローカル HTTP サーバ
    start-motion-book.ps1 … 起動スクリプト (PowerShell)
    stop-motion-book.ps1  … 停止スクリプト (PowerShell)
  start-motion-book.bat   … 起動 (バッチファイル)
  stop-motion-book.bat    … 停止 (バッチファイル)
```

## 使い方

ローカルサーバを起動して、ブラウザで絵本を開く構成です。

### いちばん簡単

`start-motion-book.bat` を実行します。

- 必要なら自動で `vite build` を実行します
- ローカルサーバを起動します
- 既定のブラウザで絵本を開きます

### 停止

`stop-motion-book.bat` を実行します。

## npm から実行する場合

```bash
npm run start:local
```

停止:

```bash
npm run stop:local
```

## 前提

- Windows
- Node.js / npm がインストール済み

## 補足

- ローカルサーバは `127.0.0.1` の空きポートを自動で使います
- すでに起動中なら、そのサーバを再利用してブラウザだけ開きます
